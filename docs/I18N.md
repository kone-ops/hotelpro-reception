# Internationalisation (i18n) – Système bilingue FR/EN

## Vue d’ensemble

L’application utilise le système de traduction de Laravel pour un affichage en **français** et **anglais**. **La langue s’applique partout** : le middleware `SetLocale` est exécuté sur **toutes les requêtes web** (dashboard, super admin, réception, hôtel, formulaires publics, etc.), donc l’interface affiche la même langue sur toute l’application.

**Le français est la langue par défaut partout** : au premier accès (sans choix de langue), l’application affiche le français ; en cas de traduction manquante (par ex. en anglais), le français est utilisé comme langue de repli.

## Structure des fichiers de langue

- **`lang/`** (à la racine du projet)
  - **`fr.json`** / **`en.json`** : clés courtes pour les chaînes déjà utilisées en `__('Texte')` (ex. Dashboard, Profile, Log Out).
  - **`fr/`** et **`en/`** : fichiers PHP par domaine :
    - **`common.php`** : libellés génériques (boutons, actions, langue).
    - **`modules.php`** : libellés des modules (housekeeping, laundry).
    - **`super.php`** : interface Super Admin (hôtels, modules, etc.).
    - **`reception.php`** : réception (tableau de bord, fiche de police).
    - **`design.php`** : champs du formulaire (design hôtel).

## Configuration

- **`config/app.php`**
  - `locale` : **français (`fr`)** par défaut — toute l’application s’affiche en français si l’utilisateur n’a pas choisi l’anglais.
  - `fallback_locale` : **français (`fr`)** — si une clé manque (ex. en anglais), on affiche le français. Ainsi le français « prend partout ».
  - `supported_locales` : `['fr', 'en']`.

- **`.env`** (optionnel)
  - `APP_LOCALE=fr`
  - `APP_FALLBACK_LOCALE=fr`

## Changement de langue (directement dans l’interface)

- **Route** : `GET /locale/{locale}` (ex. `/locale/en`), nommée `locale.switch`.
- Comportement : enregistrement de la locale en session puis redirection vers la page précédente.
- **Où l’utilisateur change la langue** :
  - **Connecté** : dans la **topbar**, bouton dropdown « Langue : Français » / « Langue : English » (icône globe) ; au clic, choix « Français » ou « English », la page se recharge dans la langue choisie.
  - **Non connecté** (login, register, etc.) : liens **Français | English** en haut à droite du layout guest.

## Bonnes pratiques

1. **Vues Blade**  
   Utiliser `__('clé')` ou `@lang('clé')` pour tout texte affiché à l’utilisateur.  
   - Clés courtes (sans point) : cherchées dans `lang/{locale}.json`.  
   - Clés avec point (ex. `super.hotels.list`) : cherchées dans `lang/{locale}/super.php` sous `['hotels']['list']`.

2. **Contrôleurs / PHP**  
   Utiliser `__('modules.housekeeping.label')`, `trans('common.save')`, etc., pour les libellés renvoyés à la vue ou en JSON.

3. **Nouveaux textes**  
   - Ajouter la clé dans le fichier PHP du domaine (ex. `super.php`) ou dans `fr.json` / `en.json`.  
   - Traduire dans les deux langues pour éviter les chaînes manquantes.

4. **Données métier**  
   Les noms d’hôtels, chambres, etc. restent en base dans la langue saisie ; seuls les libellés de l’interface sont traduits.

## Étendre à d’autres langues

1. Créer `lang/{code}/` (ex. `lang/es/`) et y copier/adapter les fichiers PHP (common, modules, super, etc.).
2. Créer `lang/{code}.json` avec les mêmes clés que `fr.json` / `en.json`.
3. Ajouter le code dans `config/app.php` → `supported_locales`.
4. Optionnel : ajouter une colonne `locale` sur la table `users` et une méthode `getPreferredLocale()` sur le modèle User pour persister la préférence.

## Middleware – application sur toute l’app

Le middleware **`App\Http\Middleware\SetLocale`** est enregistré sur le groupe **`web`** : il s’exécute sur **toutes les pages** (login, dashboard, super admin, réception, hôtel, housekeeping, laundry, formulaires, etc.). La langue choisie (ou le français par défaut) s’applique donc **partout** dans l’application. Il définit `App::setLocale()` selon : session → préférence utilisateur (si définie) → Accept-Language → défaut (`fr`).
